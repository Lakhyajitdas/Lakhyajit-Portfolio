
# Contributing Guidelines

Thanks for your interest in contributing to this project! :tada:

## How Can I Contribute?
- **Bug reports**: Use the issue template and include steps to reproduce, expected behavior, and screenshots/logs where helpful.
- **Feature requests**: Open an issue describing the use case and proposed solution.
- **Documentation**: Fix typos, improve explanations, or add examples.
- **Code contributions**: Fork the repo, create a feature branch, and open a pull request (PR).

## Development Workflow
1. Fork the repository and clone your fork.
2. Create a new branch:
   ```bash
   git checkout -b feature/your-feature-name
   ```
3. Make your changes and add tests if applicable.
4. Run linters/tests locally.
5. Commit your changes using conventional-style messages:
   ```
   feat: add portfolio section for projects
   fix: correct navbar link
   docs: update README with setup steps
   ```
6. Push your branch and open a PR against `main`.

## Pull Request Checklist
- PR description explains **what** and **why**.
- Linked issue (if any) is referenced (e.g., `Closes #123`).
- Tests updated/added (if applicable).
- No unrelated changes in the same PR.

## Code Style
- Use meaningful names and comments where necessary.
- Keep commits small and focused.
